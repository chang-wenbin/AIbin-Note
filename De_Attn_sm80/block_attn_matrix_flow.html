关键优化总结： (1) Q 只在 n=0 时从 SMEM 加载到 Register，后续复用（A_in_regs=true）； (2) K 和 V 的加载与计算异步重叠（cp_async + software pipeline）； (3) P 从 C-layout 经 SMEM 中转到 A-layout 时复用 sQK 空间； (4) O 写出时复用 sQ 空间； (5) Paged KV Cache 用增量指针跳转（block_table 差值），避免重算绝对地址； (6) Online softmax 使递推更新成为可能，每个 partition 独立计算后由 reduce kernel 合并。
